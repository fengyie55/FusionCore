# FusionCore 平台阶段协调说明

## 1. 目的

本文用于记录当前平台底座模块的阶段协调结果，便于后续按治理阶段推进时统一参考。

本文只描述阶段状态与协调结论，不定义实现细节。

## 2. 协调范围

本次协调只覆盖以下模块：

- FusionKernel
- FusionConfig
- FusionLog
- FusionUI

说明：

- 这些模块是当前平台底座的主要接线点
- 它们已经具备继续收敛到稳定最小闭环的基础
- 本次协调不推进 `FusionData`

## 3. 阶段协调结果

当前协调结果如下：

| 模块 | 协调后阶段 | 说明 |
| --- | --- | --- |
| FusionKernel | P2→P3 | 已形成宿主、运行时、模块生命周期与最小闭环接线 |
| FusionConfig | P2→P3 | 已形成配置节、Snapshot、Provider、Composite Source 与最小默认装配边界 |
| FusionLog | P2→P3 前半段 | 已形成日志语义、日志上下文、writer 组合、文件 writer 与配置接线前半段 |
| FusionUI | P2→P3 前半段 | 已形成 WPF Shell、导航骨架、状态展示、运行态摘要入口与日志入口只读模型 |

## 4. 这里的“推进到 P2”是什么意思

这里的“推进到 P2”指的是：

- 已排除仍停留在占位或纯规划层的主底座模块
- 已形成最小可接线、可测试、可继续演进的语义骨架
- 已避免把底座模块提前做成完整平台框架

它不表示：

- 所有模块都必须停在 `P2`
- 已经进入完整运行时或完整业务实现
- 可以忽略各模块的治理阶段差异

## 5. 互相依赖的边界

- `FusionKernel` 提供宿主与运行时边界，供 `FusionConfig`、`FusionLog` 和 `FusionUI` 接入
- `FusionConfig` 提供配置快照与读取边界，供宿主与表现层消费
- `FusionLog` 提供日志语义与 writer 边界，供宿主与表现层消费
- `FusionUI` 只消费只读摘要和投影，不承载业务真相

## 6. 当前不做什么

当前协调不做以下内容：

- 不推进 `FusionData` 进入 `P2`
- 不引入数据库、ORM、缓存或消息中间件
- 不引入 IPC、协议实现、调度算法或设备控制逻辑
- 不把任何模块扩展成大而全框架

## 7. 后续建议

后续推进顺序建议保持为：

1. 维持 `FusionKernel`、`FusionConfig`、`FusionLog`、`FusionUI` 的稳定闭环
2. 再按治理计划逐步处理其余核心模块
3. `FusionData` 仍保持 `P1` 规划层，不在本轮协调内推进
