# FusionStudio 信息架构（E95 对齐，P1）

## 1. 架构目标

`FusionStudio` 的信息架构目标是：在不引入业务控制逻辑的前提下，为工程与调试角色提供稳定、可扩展的工作台入口。

## 2. 角色视角

面向角色：
- 开发工程师
- 现场调试工程师
- 平台支持工程师

不面向：
- 设备生产操作员的主操作流程（该职责属于 `FusionUI`）

## 3. 最小布局分区

建议采用四区布局：
- 顶部：平台状态摘要区
- 左侧：工程导航区
- 中央：工作区（只读摘要页/占位页）
- 底部：消息与提示区

## 4. 最小导航结构

一级导航建议：
- `Overview`（工程总览）
- `Configuration`（工程配置入口）
- `Logs`（详细日志入口）
- `Runtime Diagnostics`（运行诊断入口）
- `Debug Assistant`（调试助手入口）
- `Module Explorer`（模块状态入口）

说明：
- 这些入口当前只用于承载页面骨架与只读摘要模型。
- 不在本阶段承载真实执行逻辑。

## 5. 页面语义约束

### 5.1 Overview
- 展示宿主状态、profile、runtime root 摘要。
- 展示最小模块状态汇总。

### 5.2 Configuration
- 展示配置入口说明与映射结果摘要。
- 不实现配置编辑与保存。

### 5.3 Logs
- 展示日志详细入口摘要与样例列表。
- 不实现搜索、过滤引擎、实时流框架。

### 5.4 Runtime Diagnostics
- 展示宿主与模块运行诊断摘要。
- 不实现健康检查平台或自动修复逻辑。

### 5.5 Debug Assistant
- 仅作为调试能力入口占位。
- 不实现调试命令执行引擎。

### 5.6 Module Explorer
- 展示模块标识、状态与摘要。
- 不直接操控模块生命周期。

## 6. 与接线边界的关系

`FusionStudio` 页面数据输入应来自：
- `FusionApp` 提供的 bootstrap 结果
- `FusionKernel` 提供的运行摘要
- `FusionConfig` 提供的 section 映射结果
- `FusionLog` 提供的日志摘要投影

不允许页面直接持有后台内部真相对象。

## 7. 当前阶段不做什么

不做：
- 业务页功能落地
- 配置编辑与写回
- 日志平台能力
- 调试执行器
- 跨进程交互实现

## 8. P2 最小落地方向

进入 P2 时优先落地：
1. Shell 与导航骨架
2. 只读摘要模型接线
3. 页面占位与状态展示
4. 基于 `FusionApp` 的最小 bootstrap 入口
